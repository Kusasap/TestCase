TestCase API — Django + DRF + PostgreSQL + Docker

Этот проект представляет собой REST API для работы с вопросами и ответами.

**Реализовано:**

CRUD для вопросов
Добавление ответов к вопросу
Получение и удаление ответа
Логирование всех действий в файл
Запуск в Docker 
Тесты на pytest

**Функционал API**

Вопросы
Метод	URL	Описание
GET	/questions/	Получить список вопросов (без ответов)
POST	/questions/	Создать новый вопрос
GET	/questions/{id}/	Получить вопрос + ответы
DELETE	/questions/{id}/	Удалить вопрос и все ответы

Ответы
Метод	URL	Описание
POST	/questions/{id}/answers/	Добавить ответ на вопрос
GET	/answers/{id}/	Получить ответ
DELETE	/answers/{id}/	Удалить ответ

**Технологии**

Python 3.12.1
Django 5.2.8
Django REST Framework 3.16.1
PostgreSQL 17
Docker + Docker Compose
pytest 8.4.1

**Запуск программы**

Для старта программы необходимо создать postgresql базу данных со следующими параметрами:
Имя базы данных: django
Имя администратора: admin
Пароль администратора: admin

После этого в powershell необходимо прописать

docker compose up --build
PostgreSQL стартует автоматически → Django мигрирует → сервер запустится.
API будет доступен по адресу:
http://127.0.0.1:8000/

**Логирование**

Все запросы и действия пишутся в файл:
logs.log

Формат логов:
[2025-11-28 17:49:02,360] INFO project GET /questions – получен список вопросов
